# 智能电影数据分析与推荐平台 - 产品需求文档 (PRD)

## 1. 项目概述
* **产品定位**：一款驱动于用户行为数据与推荐算法的个性化电影分发与决策支持平台。
* **目标用户**：面临海量影视资源“选择困难症”、需要精准内容推荐的C端观影群体；以及需要监控平台数据指标的B端运营人员。
* **核心痛点**：
    1. **信息过载**：传统分类检索效率低，用户难以快速定位感兴趣的影片。
    2. **长尾内容沉没**：优质冷门电影缺乏曝光渠道，头部效应过强。
    3. **冷启动困难**：新用户注册后缺乏历史数据，难以立即获得精准推荐。

## 2. 核心数据指标定义 (Metrics)
为衡量本平台推荐算法的业务价值，设定以下北极星指标及过程指标：
* **CTR (点击率)**：推荐模块中，影片被点击的次数 / 影片曝光总次数。用于衡量推荐列表的直接吸引力。
* **留存率 (Retention)**：次日/7日/30日留存率，验证精准推荐是否有效提升了用户粘性。
* **推荐覆盖率 (Coverage)**：周期内被推荐出来的电影数量 / 电影库总数量。用于评估长尾电影的挖掘效果。
* **冷启动转化时长**：新用户完成兴趣标签勾选，到产生首次“评分/收藏”行为的平均时间间隔。

## 3. 核心功能详情：数据可视化看板 (B端)
### 3.1 业务场景
数据产品经理及运营人员需要通过直观的可视化图表，实时监控平台流量健康度，并对比不同推荐策略（如：协同过滤 vs 基于内容）的实际转化效果，以此作为迭代依据。

### 3.2 功能需求列表
| 功能模块 | 功能描述 | 交互规则与展现形式 | 优先级 |
| :--- | :--- | :--- | :--- |
| **全站流量概览** | 统计展示核心大盘数据。 | 顶部卡片式展示：日活(DAU)、新增用户数、总播放/点击量。数值需带有环比昨日/同比上周的升降箭头标识。 | P0 |
| **推荐转化漏斗** | 追踪用户从“曝光->点击->详情页停留->收藏/评分”的转化链路。 | 漏斗图呈现。支持按日期区间筛选，需明确展示每一步的流失率。 | P0 |
| **热词与标签分布** | 展示当前站内最受关注的电影标签和搜索热词。 | 词云图展现，字体大小与搜索频率成正比。支持点击词汇反查相关影片列表。 | P1 |
| **算法效果对比** | 对比A/B测试下，不同推荐算法的各项业务指标差异。 | 双线折线图，X轴为时间，Y轴为CTR或平均停留时长，直观展示策略优劣。 | P1 |

### 3.3 数据约束与规则
* 看板数据更新频率为 `T+1`（离线计算）或准实时（延迟<5分钟）。
* 数据权限：仅对拥有 `Admin` 或 `Data_Analyst` 角色的后台账号开放。
